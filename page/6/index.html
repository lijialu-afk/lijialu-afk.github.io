Html

HTML 标题
HTML 标题（Heading）是通过 h1 标签来定义的。

HTML 段落
HTML 段落是通过标签 p 来定义的。

HTML 链接
HTML 链接是通过标签 a 来定义的。

提示:在 href=链接的地址，后面的文字是代替链接的文字。
HTML 图像
HTML 图像是通过标签 img 来定义的.

提示： 图像的名称和尺寸是以属性的形式提供的。
定义图像的语法是：
《img src=”url” /》（使用英文字符，url可以是路径也可以是链接）
具体内容详见（https://www.w3school.com.cn/html/html_images.asp）

插入本地图片方法

绝对路径
比如，你的图片存放在D盘images目录里，名称是img.png。那么绝对路径应该是这样：D:\images\img.png。
相对路径
如果，你的html文件存放的位置跟图片一样，那么就直接表达：img.png。
子路径
那如果，你的html文件存放在D盘里，那么就是子路径了：images/img.png。
父路径
html文件存放在D盘里的images文件里的css文件中，那么就使用父路径了：../img.png，当然了，也可以../../images/img.png。

Php变量覆盖2

今天写了本周web小分队的最后一次web题，题目如下：

由题可知，这是一道$$导致的变量覆盖问题。

题目分析：

源码包含了flag.php文件，并且需要满足3个if里的条件才能获取flag，题目中使用了两个foreach并且也使用了$$.两个foreach中对 $$key的处理是不一样的，满足条件后会将$flag里面的值打印出来，所以$flag是在flag.php文件文件中的。 由于flag值被覆盖掉了，所以要先将flag值赋给变量_200或变量_403,然后利用die（200）或die（403）将flag显示出来。

解题方法：

由于flag的值被覆盖掉，所以只能将$flag得知赋给$_200,通过die（_200）将flag输出来。

最终payload如下：

GET：?_200=flag

POST：flag=hahaha（flag=任意字符）

成功得到flag~

注； $$ 导致的变量覆盖问题在CTF代码审计题目中经常在foreach中出现，使用foreach来遍历数组中的值，然后再将获取到的数组键名作为变量，数组中的键值作为变量的值。因此就产生了变量覆盖漏洞。如：get ?name=test 会将$name的值覆盖，变为test。 （aha~又是正经的一天~）

今天做了学长给的一道php变量覆盖题。学长给的web题

仔细一看，就，变整齐了？
再一看，根据学长所说，这是一道php变量覆盖题，借助百度，了解一下，就是让两变量相等
输入payload：【get】？liwu=&content 开始执行

duang成功得到flag~
注：extract()语法： extract(array,extract_rules,prefix)

(一)


extract() 函数从数组中将变量导入到当前的符号表。
该函数使用数组键名作为变量名，使用数组键值作为变量值。针对数组中的每个元素，将在当前符号表中创建对应的一个变量。
第二个参数 type 用于指定当某个变量已经存在，而数组中又有同名元素时，extract() 函数如何对待这样的冲突。
该函数返回成功导入到符号表中的变量数目。
如果变量liwu和变量content的值相同，就会输出flag
所以构造 ?liwu=&content flag就可以出来。

(二）


题目使用了extract($_GET)接收了GET请求中的数据，并将键名和键值转换为变量名和变量的值，然后再进行两个if 的条件判断，所以可以使用GET提交参数和值，利用extract()对变量进行覆盖，从而满足各个条件。
解题方法：
GET请求 ?flag=&gift=，extract()会将flag和flag和gift的值覆盖了，将变量的值设置为空或者不存在的文件就满足gift==gift==content。
最终PAYLOAD：GET DATA： ?flag=&gift= 也可以得到flag。
